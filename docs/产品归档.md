# BN-Trader 当前版本产品归档

适用版本：v1.0.0

记录当前版本已经交付的产品能力、验证基线、明确边界和后续工作，作为发布、升级和客户支持的共同参照。

## 1. 当前交付基线

- Apple 风格明亮/暗色桌面界面，侧栏采用线性图标，悬停使用浅银灰反馈。
- 五类行情场景、策略参数、资金池、持仓、三层风险设置、手动下单和交易日志。
- 自动化投资任务台、独立币种资金分配表、任务手动启停和重启后不自动恢复。
- 默认支持 BTC、ETH、BNB、SOL、XRP 与 USDC 的 USDT 交易对；USDC/USDT 使用稳定币价格精度。
- K 线自适应价格精度、右侧价格轴、实时价格标签和当前价虚线。
- 日志支持鼠标与键盘选择、复制按钮、右键菜单、多行复制及 Ctrl+C / Command+C。
- 顶部 API 设置入口、左侧帮助中心与 AI 研报建设中占位页。
- 策略、全局参数、API、自动化任务和资金比例使用稳定用户目录持久化。

## 2. 产品边界与已知待办

- AI 研报当前仅为建设中页面，尚不生成研究结论或投资建议。
- 简体中文之外的多语言界面尚未完成，不能作为当前版本能力对外承诺。
- 当前没有应用内自动下载、静默升级、安装包正式签名和 macOS 公证。
- 日交易次数、连续亏损、冷静期和日累计亏损已有判断逻辑，但真实成交结算回写仍待完整接入。
- API 设置页的账户连接测试不能替代 TRADE 权限测试；真实下单前仍需确认现货交易权限与 IP 白名单。
- 当前自动化标的只接受 USDT 计价交易对；USDC/USDT 已支持，USDC/USD 未纳入当前产品。

## 3. 文档与质量基线

- docs/help_content.json 是产品资料、产品手册、系统帮助、升级文档和当前版本归档的统一内容源。
- 应用内帮助中心直接读取统一内容源，F1 可打开并搜索。
- tools/generate_help_docs.py 负责生成全部 Markdown 产品文档。
- GitHub Actions 在主分支更新后自动检查并提交生成文档。
- 发布前必须通过单元测试、JSON 校验、文档生成一致性检查和 git diff 格式检查。

## 4. 下一阶段建议

- P0 交易安全：保留 Binance JSON 错误码，在 API 设置页使用 /api/v3/order/test 验证真实 TRADE 权限，并明确区分正式网与测试网。
- P0 风控闭环：把真实成交与平仓结果完整回写到日交易次数、日累计亏损、连续亏损、冷静期和强制复盘状态。
- P0 订单合规：下单前读取交易所 PRICE_FILTER、LOT_SIZE、MIN_NOTIONAL 等规则，统一修正价格与数量精度。
- P1 自动化可靠性：增加订单幂等标识、启动前检查、未完成订单对账、任务级暂停原因和异常恢复策略。
- P1 发布工程：完成 Windows 签名、macOS Developer ID 签名与公证、安装包校验值、数据库备份和可验证回滚。
- P2 产品扩展：在核心交易与升级链路稳定后，再完成八种语言和 AI 研报；研报内容不得直接绕过风控触发订单。
